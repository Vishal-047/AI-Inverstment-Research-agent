SECTION 1 — WHAT WAS BUILT
--------------------------------------------------------------------------------

A full-stack AI agent that researches startup founding teams and returns an
investment verdict: Invest / Pass with note / Pass / Inconclusive.

Core philosophy: "Back the founder, not just the market."

Tech stack:
  - Next.js 16.2.10 (App Router, TypeScript, Tailwind CSS)
  - LangGraph.js (StateGraph with iterative retry loop)
  - Google Gemini 2.0 Flash (via @langchain/google-genai)
  - Tavily Search API (free tier web search)
  - Zod (structured data extraction)
  - Deployed target: Vercel

SECTION 3 — AGENT ARCHITECTURE
--------------------------------------------------------------------------------

The agent uses an Iterative StateGraph:

  START → planQueriesNode → executeSearchNode → extractDataNode
                ↑                                      ↓
                └─── [if dataConfidence=low AND retryCount<2] ←──┘
                                                       ↓
                                              scoreCompanyNode
                                                       ↓
                                             decideOutcomeNode → END

NODES:
  1. planQueriesNode   — Gemini generates 3-4 targeted founder-focused queries
                         On retry: reads missingInformation and generates new queries
  2. executeSearchNode — Tavily runs each query (basic depth, 3 results each)
                         Each result truncated to 800 chars to prevent Gemini loop errors
  3. extractDataNode   — Gemini extracts structured data into Zod schema
                         Total context capped at 8000 chars
  4. scoreCompanyNode  — PURE TYPESCRIPT (no LLM) applies weighted rubric
  5. decideOutcomeNode — Gemini writes final VC memo narrative (prose only)

SCORING RUBRIC (pure TypeScript, deterministic):
  Track Record  (30%) → 9 (exits found) / 7 (ventures, no shutdowns) / 4 (shutdowns) / 3 (none)
  Domain Fit    (25%) → 8 (strong background) - 3 if pivot flag detected
  Team Velocity (25%) → 8 (≥2 hires) / 5 (1 hire) / 3 (0 hires)
  Conviction    (20%) → 8 (stable) / 6 (unclear) / 2 (departures found)

DECISION RULES:
  Invest          → weighted score ≥ 7.0 AND no single category < 4
  Pass with note  → weighted score 5.5–7.0
  Pass            → weighted score < 5.5 OR any single category < 4 (floor rule)
  Inconclusive    → dataConfidence="low" OR missingInformation.length > 3

SECTION 5 — ALL ERRORS ENCOUNTERED AND FIXES
--------------------------------------------------------------------------------

ERROR 1: Module-level LLM initialization failing
  Message: "Please set an API key for Google GenerativeAI in GOOGLE_API_KEY"
  Root cause: `new ChatGoogleGenerativeAI()` was called at module import time,
              before Next.js/Turbopack had injected env vars from .env.local
  Fix: Moved LLM construction into a `getLlm()` factory function called inside
       each node handler at request time (not at import time)

ERROR 2: Wrong env var name
  Message: "GOOGLE_API_KEY is not set" (our custom error)
  Root cause: .env.local file had `GEMINI_API_KEY` but library reads `GOOGLE_API_KEY`
  Fix: Standardised on `GOOGLE_API_KEY` (what the library expects by default)

ERROR 3: .env.local not saved to disk
  Message: Same "GOOGLE_API_KEY is not set" persisting after restarts
  Root cause: File was edited in VS Code but Ctrl+S was never pressed — disk
              still had the original empty template. Diagnosed with `cat -A`.
  Fix: User saved the file; also recreated it programmatically to be safe

ERROR 4: Gemini model 404 Not Found
  Message: "models/gemini-1.5-flash is not found for API version v1beta"
  Root cause: Account created July 2026 — gemini-1.5-flash is deprecated/removed
  Fix: Queried the models API to list available models:
       curl "https://generativelanguage.googleapis.com/v1beta/models?key=KEY"
       Switched to gemini-2.0-flash-001, then gemini-3.5-flash

ERROR 5: Rate limit (quota = 0) on gemini-2.0-flash
  Message: "429 Too Many Requests — free_tier_requests limit: 0"
  Root cause: New accounts have zero free-tier quota for older gemini-2.0 models;
              Google is pushing new users to latest generation models
  Fix: Switched to gemini-3.5-flash (confirmed available for this account)

ERROR 6: Gemini "looping content" detection
  Message: "model output error: Your output is flagged for looping content"
  Root cause: Large/repetitive web search content was sent to Gemini structured
              output; the model flagged its own response as looping
  Fix A: Truncated each search result content to 800 chars per result
  Fix B: Capped total extraction context at 8000 chars

ERROR 7: LangGraph cycle detection
  Message: "There was a problem — looping content"
  Root cause: LangGraph's built-in cycle detector fires when graph revisits a node
              Our retry loop is intentional, not an infinite cycle
  Fix: Added `recursionLimit: 25` to the `investmentAgentGraph.invoke()` call
       Math: max 3 passes × 5 nodes = 15 steps < 25 limit

ERROR 8: TypeScript errors — Annotation API change
  Message: "Property 'value' is missing in type '{ default: () => null }'"
  Root cause: Newer @langchain/langgraph version changed the Annotation API.
              Previously: `Annotation<T>({ default: () => value })` was valid
              Now: `value` (reducer function) is REQUIRED alongside `default`
              Also: `reducer` key was renamed to `value` in searchResults
  Fix: Added `value: (_, update) => update` to all simple Annotations
       Changed `reducer:` key to `value:` in searchResults accumulator

ERROR 9: TypeScript error — initialState type mismatch
  Message: "Type 'null' is not assignable to extractedData"
  Root cause: Passing null for fields that have defaults caused type error
              because the UpdateType doesn't accept null for typed fields
  Fix: Simplified initialState to only pass fields without defaults
       (companyName + queries), let LangGraph use Annotation defaults for rest

SECTION 7 — KNOWN LIMITATIONS (TO IMPROVE)
--------------------------------------------------------------------------------

  1. No streaming — UI shows "researching..." for 60-90s with no live updates
     Improvement: Replace fetch() with SSE (Server-Sent Events) for step-by-step
     updates as each node completes

  2. Scoring heuristics are simple thresholds, not continuous
     e.g. "1 hire found = 5" vs "5 hires found = still 8"
     Improvement: Graduate the scoring scale, reward quantity of evidence

  3. Co-founder departure rule is too blunt
     Any departure = score 2, regardless of timing or context
     Improvement: Penalise early departures (<2 years) heavily, reduce penalty
     for later transitions

  4. No disambiguation for ambiguous company names
     Searching "Apple" finds Apple Inc., not a seed-stage startup named Apple
     Improvement: Ask user for city/sector context when company name is common

  5. Search results may return PR/marketing content, not real founder data
     Improvement: Add LinkedIn-specific query variants, filter by domain
     (linkedin.com, crunchbase.com, techcrunch.com)

  6. The agent caches nothing — same company searched twice = full API cost twice
     Improvement: Add Redis or in-memory cache keyed by company name

  7. Vercel deployment not yet done
     Next step: `vercel deploy` from /home/vishal/Desktop/Agent
